üìÑ HTML Basics

1. Document Structure

What it is: the "skeleton" of every webpage.

Why it matters: Browsers need a predictable wrapper to know how to parse and display your content.

How it works:

<!DOCTYPE html>           <!-- tells browser: HTML5 -->
<html lang="en">          <!-- root element, lang for accessibility/SEO -->
  <head>                  <!-- metadata (title, styles, character set) -->
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Page Title</title>
  </head>
  <body>                  <!-- everything visible goes here -->
    <!-- your content -->
  </body>
</html>

Visual tip: imagine nesting Russian dolls‚Äî<html> wraps <head> and <body>, which wrap everything else.

2. Text & Headings

What it is: semantic tags for titles and paragraphs (<h1>‚Äì<h6>, <p>).

Why it matters: Screen-readers & search engines rely on heading hierarchy to understand your page.

How it works:

<h1>Main Title</h1>
<p>This is a paragraph of text explaining something under the main title.</p>
<h2>Subsection</h2>
<p>More detail here.</p>

Visual tip: think of headings like chapters and sub-chapters in a book.

3. Links & Images

What it is: anchors (<a>) and media embeds (<img>).

Why it matters: navigation and visual content make sites engaging and accessible.

How it works:

<a href="https://example.com" target="_blank" rel="noopener">
  Visit Example
</a>
<img src="photo.jpg" alt="A smiling cat" width="300">
  • href: destination URL
  • target="_blank": open in new tab
  • rel="noopener": security best practice
  • alt: describes image for users who can't see it

Visual tip: picture each <a> as a bridge‚Äîyou label it and point it where you want visitors to go.

4. Lists

What it is: ordered (<ol>), unordered (<ul>) and definition lists (<dl>).

Why it matters: grouping related items semantically for clarity and accessibility.

How it works:

<ul>
  <li>First bullet</li>
  <li>Second bullet</li>
</ul>

<ol>
  <li>Step one</li>
  <li>Step two</li>
</ol>

<dl>
  <dt>HTML</dt><dd>HyperText Markup Language</dd>
  <dt>CSS</dt><dd>Cascading Style Sheets</dd>
</dl>

Visual tip: imagine a grocery list (<ul>) vs. recipe steps (<ol>).

5. Forms & Inputs

What it is: <form> wrapper + <input>, <textarea>, <select>, <button>.

Why it matters: the primary way users send you data (emails, orders, comments).

How it works:

<form action="/submit" method="POST">
  <label for="email">Email:</label>
  <input type="email" id="email" name="email" required>

  <label for="message">Message:</label>
  <textarea id="message" name="message"></textarea>

  <button type="submit">Send</button>
</form>
  • action: where data goes
  • method: GET vs. POST
  • required: browser-enforced validation

Visual tip: see each input as a field on a paper form‚Äîlabel next to the blank.

6. Semantic Elements

What it is: meaningful wrappers: <header>, <nav>, <main>, <section>, <article>, <aside>, <footer>.

Why it matters: improves SEO, accessibility, and maintainability.

How it works:

<header>‚Ä¶</header>

  <article>‚Ä¶</article>
  <aside>‚Ä¶</aside>

Visual tip: treat them like zones on a map: header at top, nav on side, footer at bottom.

7. Media

What it is: <audio>, <video>, <figure> + <figcaption>.

Why it matters: native playback & captions without plugins.

How it works:

<figure>
  <video controls width="600">
    <source src="movie.mp4" type="video/mp4">
    Your browser doesn’t support HTML5 video.
  </video>
  <figcaption>My vacation clip</figcaption>

Visual tip: picturing a photo album: each <figure> is a framed picture with a caption.

8. Tables

What it is: tabular data: <table>, <tr>, <th>, <td>.

Why it matters: best for spreadsheets, schedules, or any grid of data.

How it works:

<table>
  <caption>Monthly Sales</caption>
  <tr>
    <th>Product</th><th>Qty</th><th>Revenue</th>
  </tr>
  <tr>
    <td>Widget A</td><td>100</td><td>$1,000</td>
  </tr>
</table>

Visual tip: imagine an Excel sheet embedded in your page.

üé® CSS Fundamentals

1. Selectors & Specificity

What it is: how you "point" at elements to style them.

Why it matters: determines which rule wins when multiple apply.

How it works:

/* Type selector‚Äîtargets all <p> */
p { color: blue; }

/* Class selector‚Äîtargets elements with class="btn" */
.btn { padding: 8px; }

/* ID selector‚Äîtargets one unique element with id="main" */
#main { margin: 0 auto; }

/* Specificity ranking: inline > ID > class/attribute/pseudo-class > element/pseudo-element */

Visual tip: think of CSS rules as arrows‚Äîsome arrows (inline) are stronger than others (type).

2. Box Model

What it is: every element is a rectangle with layers: content ‚Üí padding ‚Üí border ‚Üí margin.

Why it matters: controls spacing and sizing.

How it works:

.box {
  width: 200px;       /* content width */
  padding: 20px;      /* inside buffer */
  border: 5px solid;  /* visible edge */
  margin: 10px;       /* outside gap */
}

/* [ margin | border | padding | content ] */

Visual tip: picture a gift box‚Äîwith padding as bubble wrap, border as the box itself, margin as space between boxes.

3. Typography & Colors

What it is: fonts, sizes, line-height, color properties.

Why it matters: readability and brand identity.

How it works:

body {
  font-family: 'Arial', sans-serif;
  color: #333;              /* text color */
  background-color: #f4f4f4;
  line-height: 1.6;         /* spacing between lines */
}
h1 { font-size: 2rem; }     /* rem = relative to root font-size */

Visual tip: see text as painting on a canvas‚Äîcolor & spacing set the mood.

4. Layout

a) Flexbox (1-dimensional row or column).

.container {
  display: flex;              /* turn on flex mode */
  justify-content: center;    /* main-axis alignment */
  align-items: stretch;       /* cross-axis alignment */
  flex-wrap: wrap;            /* allow wrapping */
}

b) Grid (2-dimensional rows and columns).

.grid {
  display: grid;
  grid-template-columns: 1fr 2fr;  /* two columns: 1 part + 2 parts */
  gap: 10px;                        /* space between cells */
}

Visual tip: imagine graph paper‚Äîyou define rows & columns and place items.

5. Responsive Design

What it is: making your site work on phones ‚Üí desktops.

Why it matters: most traffic is mobile!

How it works:

/* base (mobile-first) */
.container { padding: 10px; }

/* larger screens */
@media (min-width: 768px) {
  .container { padding: 20px; }
}

Visual tip: imagine your layout like liquid that flows and reshapes at different widths.

6. Positioning

What it is: removes elements from normal flow or tweaks placement.

Types: static | relative | absolute | fixed | sticky.

.relative-box { position: relative; top: 10px; }
.absolute-box { position: absolute; right: 0; bottom: 0; }

Visual tip: think of marking on a transparent sheet overlaying your page.

7. Visual Effects

What it is: smooth transitions, keyframe animations, transforms.

How it works:

/* transition */
.button {
  transition: background-color 0.3s ease;
}
.button:hover {
  background-color: #ff0;
}

/* keyframe animation */
@keyframes slide-in {
  from { transform: translateX(-100%); }
  to   { transform: translateX(0); }
}
.panel {
  animation: slide-in 0.5s forwards;
}

Visual tip: picture a flipbook‚Äîeach keyframe is a page in the flipbook.

8. Preprocessors (Sass)

What it is: tools that add variables, nesting, mixins.

Why it matters: DRY and easier theming.

Example:

$primary-color: #1E88E5;

.button {
  background: $primary-color;
  &:hover {
    background: darken($primary-color, 10%);
  }
}

Visual tip: imagine CSS with superpowers‚Äîvariables are magic placeholders.

1. Syntax & Basics

What it is: the language building blocks.

Key points:

// variables

// data types
// string, number, boolean, null, undefined, object, array

Visual tip: think of variables as labeled jars holding different ingredients.

2. Control Flow

What it is: make decisions and repeat tasks.

How it works:

if (score > 50) {
  console.log('You win!');
} else {
  console.log('Try again');
}

for (let i = 0; i < 5; i++) {
  console.log(i);
}

Visual tip: picture a flowchart with diamonds (if) and loops circling back.

3. Functions

What it is: reusable blocks of code.

How it works:

function greet(user) {
  return `Hello, ${user}!`;
}
// arrow function
const add = (a, b) => a + b;

Visual tip: treat functions like little machines‚Äîyou feed inputs and get outputs.

4. DOM Manipulation

What it is: changing the page after it's loaded.

How it works:

const btn = document.querySelector('.btn');
btn.textContent = 'Clicked!';

// create new element
const p = document.createElement('p');
p.textContent = 'New paragraph';
document.body.append(p);

Visual tip: see the DOM as a tree‚Äîeach node is a branch you can prune, graft, or rename.

5. Events

What it is: reacting to user actions.

How it works:

btn.addEventListener('click', () => {
  alert('Button was clicked!');
});
// event propagation: capture ‚Üí target ‚Üí bubble

Visual tip: imagine throwing a rock (event) into a pond; ripples go outward (bubbling).

6. Asynchronous JS

What it is: handling tasks that take time (fetching data).

How it works:

// Promises
fetch('/api/data')
  .then(res => res.json())
  .then(data => console.log(data))
  .catch(err => console.error(err));

// async/await
async function load() {
  try {
    const res = await fetch('/api/data');
    const data = await res.json();
    console.log(data);
  } catch (e) {
    console.error(e);
  }
}

Visual tip: think of Promises as promise cards‚Äîthey'll deliver the result later.

// modules (in separate files)
// export
export function foo() { ‚Ä¶ }
// import

Visual tip: see modules as LEGO blocks‚Äîyou snap pieces together instead of rewriting.

8. Debugging

What it is: finding and fixing errors.

Tools: console.log(...), console.error(...), DevTools breakpoints, ESLint.
